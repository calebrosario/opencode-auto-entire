# Changelog

All notable changes to OpenCode Auto-Entire will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [Unreleased]

### Added
- Initial release
- Memory stack checking (Entire CLI, Claude-Mem, RTK)
- Three operation modes (prompt, auto-init, silent)
- Configuration support
- OpenCode and Claude Code integration
- Platform support (macOS, Linux, Windows, WSL)
- Comprehensive documentation
- Installation scripts (bash, PowerShell)
- GitBook-style documentation structure

### Security
- No known security vulnerabilities

## [1.0.0] - 2025-02-15

### Added
- First public release
- OpenCode plugin support
- Memory stack monitoring
- Configuration system
- Documentation website

### Changed
- Moved from personal project to public release
- Updated documentation structure
- Improved error handling

### Fixed
- Initial bug fixes from personal use

## Version Schema

- **Major version (X.0.0):** Breaking changes
- **Minor version (0.Y.0):** New features, backward compatible
- **Patch version (0.0.Z):** Bug fixes, backward compatible

## Types of Changes

- `Added` - New features
- `Changed` - Changes to existing functionality
- `Deprecated` - Soon-to-be removed features
- `Removed` - Removed features
- `Fixed` - Bug fixes
- `Security` - Security vulnerabilities

## Release Process

1. Update version in `package.json`
2. Update `CHANGELOG.md`
3. Commit changes
4. Create Git tag
5. Push tag to trigger release
6. Create GitHub release
